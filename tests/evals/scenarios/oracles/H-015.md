# H-015 oracle

Classification: `not_applicable`.

Reject the monolithic end-to-end-first plan. Decompose the feature into coherent vertical increments, each proving one behavior, assumption, interface, or risk with a bounded diff and review point. Resolve the riskiest or highest-leverage contract first, then freeze stable seams before parallel consumption.

Use the cheapest trustworthy feedback ladder. Put domain permutations, state transitions, edge cases, and failures in the fast unit-test base. Add fewer integration tests for the real adapter contract. Keep end-to-end coverage to the two critical assembled journeys and run the twenty-five-minute suite only after focused proof and affected regression are green on the frozen candidate. Stop at the first failure and return to its smallest owning layer rather than continuing the broad run or changing several layers.

Require intermediate review of the new adapter and each high-coupling increment. Preserve the required final ordered end-to-end gate; fast feedback does not waive acceptance. If the inner loop cannot produce precise feedback below twenty-five minutes despite an available lower seam, treat that latency as a design or test-placement problem.

Risks: HG-10, HG-13; late invalidation, large-review error rate, slow diagnosis, and inverted test portfolio.
