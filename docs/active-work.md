## Ownership protocol

Before execution begins, Delivery must:

1. Confirm the ticket is Ready and its dependencies are satisfied.
2. Inspect the repository and relevant evidence.
3. Register one row before any implementation edit.
4. Assign one owner per file or coherent system scope.
5. Record dependencies, overlap, integration order, and return destination.
6. Update progress and blockers at meaningful transitions.
7. Clear ownership only after integration, verification, documentation, and result handoff.
