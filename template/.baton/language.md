# Ubiquitous language

Use these terms exactly. [Workflow](workflow.md) defines their states, protocol levels, and transitions.

- **Project** — the enduring company, product, or undertaking governed by one Baton control plane.
- **Repository** — the checkout where Baton is installed and implementation is verified.
- **State** — canonical JSON for approved direction, status, ownership, protocols, and record links.
- **Record** — Project-, Goal-, or Ticket-scoped Markdown linked from State.
- **Goal** — a stakeholder-visible outcome and Release boundary containing one or more Tickets.
- **Ticket** — a bounded execution unit inside one Goal. Completion, commit, or ordinary push is not a Release.
- **Brief** — intent and acceptance detail not already clear from State.
- **Decision** — an approved durable choice with its scope and rationale.
- **Report** — completion result, verification, limitations, and follow-up.
- **Evidence** — repository or runtime material that supports a claim.
- **Review** — an independent evaluation; a required human decision is a Clearance.
- **Readiness Protocol** — the evidence depth resolved for a Ticket.
- **Clearance Protocol** — the standing human-authority policy resolved for a Goal or Ticket.
- **Release** — the exact completed Goal candidate proposed for delivery or Publication.
- **Publication** — making a Release externally available.
- **Memory** — bounded reusable context that may link but never replace State, Decisions, Evidence, or Clearances.
