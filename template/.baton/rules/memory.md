# Company memory

`.baton/memory/memory.json` is editable current truth; `history.jsonl` is its value-minimized chronology. Memory is internal infrastructure. Public skills reach its privacy-filtered reads, bootstrap reconciliation, and authority-checked writes only through their matching CLI command family. Memory informs work but never replaces State, Records, Evidence, Clearances, readiness, or priority.

Every write requires actor authority, expected revision, provenance, validation, the shared Repository lock, external backup and Report Evidence, and rollback on failure. Explicit, unambiguous user statements may be confirmed; inferred personal observations remain candidates until user confirmation. Link canonical Records instead of copying them into Memory.

At a role wake, request only assignment-specific confirmed claims: at most 10 claims and 1,800 UTF-8 bytes. Load more only on explicit demand. Never auto-inject candidates, full history, old reviews, inactive personnel, secrets, credentials, sensitive personal data, or context already present in the assignment.

The user is final memory authority. Management may confirm company identity and user-approved learning. Operations records verified assignments, outcomes, revisions, and performance evidence. Consultants and Contractors submit observations, candidates, and self-reflections but cannot confirm user facts or their own performance. Internal Audit receives read-only bounded context only when memory is inside its exact evaluation boundary.

Preserve stable personnel identity and evidence-linked contextual strengths. Select coworkers by capability, evidence, availability, model quality, and required reasoning; style never weakens those requirements. Do not create universal scores, rankings, forced rotation, or novelty quotas.

Forgetting removes active claims and candidates, redacts matching local history values, refreshes views, and returns backup/rollback evidence. It never claims to rewrite Git history or erase remotes, clones, caches, repository backups, or external transaction backups.
