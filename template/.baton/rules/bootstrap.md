# Bootstrap

`$boot` starts onboarding or mature-repository adoption. The invoking task owns one continuous onboarding conversation through final intent confirmation and team assembly. It is a temporary coordinator, never permanent Management or an authority above the user.

First validate installation, provenance, state, team, and memory. Route `Needs Integration` through reviewed activation. Probe the task provider without creating anything. Native assembly requires complete list/reconciliation, create, stable identity, read, exact title write/readback, message/wake, and failed-create archival. Missing or ambiguous capability fails closed to copy-ready fallback.

Onboarding asks one question per turn: confirm the preset, capture the preferred name, resolve project identity, purpose, users, outcome, constraints, and open questions, select Readiness and Clearance Protocols, then confirm one shared summary. Look up discoverable facts; never decide user intent. Before summary confirmation, create no person, seat, task, durable project intent, Goal, or Ticket. Confirmation atomically promotes approved direction and both protocol defaults into `state/project.json` and clears its narrative draft from memory; later authority follows the selected Clearance Protocol.

After confirmation, assemble Management, Operations, then active Consultants. For each seat, persist one private creation attempt before the external action; reconcile it against the complete live inventory; record the returned stable ID and wake path; set and read back `(<project name>) - <role name> - <person name>`; then register, wake, and introduce the coworker. Never reveal an unregistered person or advance while an earlier seat or cleanup is unresolved. Contractors and Internal Audit stay disposable. Persistent goals are never used.

Resume at the first incomplete reconciled step without replaying questions, introductions, or identities. Explicit reset before completion may clear only onboarding data; it preserves project/team/non-bootstrap memory and commits cleanup records before archiving superseded tasks. Reject reset after completion.

If native assembly is unsafe, create nothing through that path and present only the next copy-ready action. Discovery remains available, but work requiring an unregistered role is `Delivery blocked`; roster state is stated as `Online` or `Awaiting task`.
