# Security policy

## Reporting a vulnerability

Do not open a public issue for a vulnerability, leaked secret, private identifier, or unsafe automation path.

Use GitHub's private vulnerability-reporting feature for this repository. Include the affected file or workflow, reproduction, potential impact, and the smallest safe remediation you know.

The maintainers will acknowledge a credible report, investigate it, and coordinate disclosure proportionally to risk. Baton is provided without warranty under the MIT License.
